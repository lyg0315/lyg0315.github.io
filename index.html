next.js 的学习路程

本文记录next.js的学习路程。
需要node>20.0.0版本

1. 快速搭建-安装 next.js-项目搭建

2.Turbopack-项目搭建

2.1 核心定位

  • 下一代打包工具：由 Webpack 原作者 Tobias Koppers 领衔开发
  • Rust 重写：基于 Rust 语言实现，充分利用原生性能优势
  • Webpack 继任者：设计目标是在功能对等的前提下，实现数量级的性能提升

2.2 核心特性

特性 说明
极速增量编译 利用 Rust 多线程和内存缓存，开发模式下代码更新以毫秒级响应
按需编译 仅编译当前页面所需的模块，而非全量构建
持久化缓存 构建结果可缓存到磁盘，重启后快速恢复
ESM & CJS 兼容 同时支持 ES Modules 和 CommonJS，无缝迁移现有项目
Tree Shaking 内置高效的死代码消除机制

性能对比
相比 Webpack，Turbopack 在大型项目中可实现：
开发启动速度：提升约 10-700 倍（取决于项目规模）
HMR 热更新：提升约 10-20 倍
与 Turborepo 的关系
Turbopack：打包工具（bundler），替代 Webpack/Vite
Turborepo：任务调度器（monorepo task runner），优化多包仓库的构建流程
两者均为 Vercel 生态工具，可配合使用但职责不同
当前状态
Next.js 默认集成：从 Next.js 13+ 开始作为可选开发服务器，Next.js 15+ 逐步成为默认
独立使用：目前主要深度集成于 Next.js，独立使用能力仍在完善中
生产构建：开发模式已成熟，生产构建能力持续迭代
适用场景
使用 Next.js 的项目（默认推荐）
对 开发体验（DX） 有极致要求的大型前端项目
希望从 Webpack 迁移且追求构建性能提升的团队
一句话总结
Turbopack 是用 Rust 重写的下一代前端打包工具，以极致的增量编译速度为核心卖点，正逐步成为 Next.js 生态的默认构建引擎。

CSS Modules 中级实战指南

CSS Modules 不是新框架，却是解决样式冲突最轻量的方案。本文不讲基础语法，聚焦项目中真正会用到的中级技巧。

1. 它到底解决了什么？

传统 CSS 的全局作用域是万恶之源——你写了一个 .button，别人的 .button 就会污染你。CSS Modules 的解法很朴素：把每个 class 名自动编译成唯一的哈希串。

本质就是编译时把 .primary → Button_primary_a3x9f，彻底杜绝冲突。

7. 与其他方案对比

方案 作用域隔离 学习成本 运行时开销 适合场景
CSS Modules ✅ 编译时 低 无 中小项目、Vite/CRA/Next.js
CSS-in-JS ✅ 运行时 中 有 高度动态的样式
Tailwind ✅ 原子类 中 无 快速原型、团队规范强
BEM 命名 ❌ 靠约定 低 无 老项目维护
Shadow DOM ✅ 浏览器原生 高 无 Web Components

CSS Modules 的最佳位置：你认可“样式写在 CSS 文件里”的传统开发方式，但想要自动的作用域隔离。它不改变你的工作流，只在编译时默默加上哈希。

一句话总结

技巧 一句话
核心原理 class 名编译成唯一哈希，杜绝冲突
composes CSS 层面的样式继承，DOM 上多类并存
:global / :local 精确控制作用域边界
CSS 变量配合 动态值用变量，静态样式用 Module
TypeScript *.module.css.d.ts 自动生成，获得智能提示
多 class 拼接 用数组 + join，或写一个 cx 工具函数

CSS Modules 不是银弹，但它是“对 CSS 改动最小的隔离方案”——如果你喜欢 .css 文件的开发体验，它几乎是最优解。
